# 2026 世界盃 體育問卷 · 成長引擎版

一個純前端、質感化的雙版本問卷，兼具兩個功能：
1. **了解會員**，找出下次活動的成功方向。
2. **會員推薦會員**（拉好友）：會員填自己的推薦碼→產生專屬邀請連結→好友填問卷並註冊投注→推薦人領階梯獎金。

## 線上網址
- 預覽（版本選擇器）：`https://mktcycy.github.io/worldcup2026-survey/`
- 會員版：`?role=member`
- 好友版（邀請連結會自動帶推薦碼）：`?role=friend&ref=推薦碼`

## 兩個版本
- **會員版**：運動偏好、體育資訊來源（獲客通路情報）、下次活動概念取捨（Conjoint）、期待型態、未滿足需求、NPS 推薦意願，最後填推薦碼並產生邀請連結。
- **好友版**：觀賽習慣、投注經驗、資訊來源、註冊誘因排序（MaxDiff）、進入顧慮、註冊意願；完成後導向 esball.net 註冊（自動帶推薦碼）。

## 拉好友獎勵邏輯（設計建議，數字由行銷定）
- 好友端：註冊＋首存 → 送免費投注金。
- 推薦人端：依單一好友累積「有效投注」給階梯 88 / 188 / 588 / 1288。
- 裂變加碼：每 3 位有效好友額外 588、月度推薦榜加碼。
- 防濫用：有效推薦＝填問卷＋註冊＋首存＋有效投注達門檻；排除互搏/超低賠率/取消注單；一人一帳、同金流/裝置/IP 不重複；30 天內達標。
- 註冊+投注追蹤與獎金發放由平台 CRM 依推薦碼撮合；問卷把「推薦人↔好友」關係寫進 Sheet 作備援對帳。

## 作答寫入 Google Sheet
前端以 `fetch(no-cors)` POST 到 Google Apps Script 網頁應用程式（見 `apps-script/Code.gs`），一題一列寫入。`分群` 欄記錄 MEMBER / FRIEND；好友版另寫「推薦人推薦碼(ref)」、會員版寫「會員自填推薦碼」。部署後把 /exec 網址填入 `js/config.js` 的 `ENDPOINT`（空=示範模式，不寫入）。

> esball.net 的實際推薦參數格式（?ref= / ?aff= / 專屬連結）請向平台確認，於 `js/questions.js` 的 `registerUrl` 調整。
