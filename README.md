# 2026 世界盃 會員問卷

一個純前端、質感化的分群問卷網頁，作答結果寫入 Google 試算表。用於 2026 世界盃活動後的會員意見調查，找出「營收未成長」的真正原因並轉化為下次活動方向。

## 線上網址
- 預覽（分群選擇器）：`https://mktcycy.github.io/worldcup2026-survey/`
- 各分群專屬連結（正式發送用）：
  - S1 有領獎＋貨量成長：`?seg=S1`
  - S2 有領獎＋貨量未成長/下降：`?seg=S2`
  - S3 有參與未領獎：`?seg=S3`
  - S4 未參與＋貨量成長：`?seg=S4`
  - S5 未參與＋貨量下降/持平：`?seg=S5`

## 分群邏輯
用後台可撈的名單發送：**活動領獎狀態 × 活動期間貨量變化**。無法從後台取得的「是否投注世足」維度，由問卷題目自答補回。

## 作答如何寫入 Google Sheet
前端以 `fetch(no-cors)` POST 到一支 **Google Apps Script 網頁應用程式**（執行於你自己的 Google 帳號、寫入你的試算表）。設定步驟見 `apps-script/Code.gs` 檔頭說明。部署後把網址填入 `js/config.js` 的 `ENDPOINT`。

> 未填 `ENDPOINT` 時為「示範模式」：流程可完整體驗，但作答不會寫入試算表（僅在瀏覽器 console 顯示）。

## 檔案結構
```
index.html          頁面骨架
css/style.css       樣式（球場夜綠×金，支援深色模式、行動裝置）
js/config.js        設定：後端網址、標題文案
js/questions.js     題庫（S1~S5，正式敬語版）
js/app.js           渲染、驗證、送出邏輯
apps-script/Code.gs  Google Sheet 後端（需自行部署）
```

## 題型
單選 / 複選 / 排序（MaxDiff，點選 1→4 名）/ 開放填答。設計原則：不問「獎金夠不夠」這類無效題，改用二選一取捨、過去行為、引導式流失原因，每題都對應一個下次活動可調整的動作。
